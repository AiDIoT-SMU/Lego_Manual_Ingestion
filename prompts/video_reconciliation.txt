You are reconciling LEGO assembly video evidence with a manual instruction file to produce
an enhanced instruction set with micro sub-steps and error corrections.

═══════════════════════════════════════════════════════════════════════
INPUT 1 — ENHANCED MANUAL (enhanced.json):
{enhanced_json}

═══════════════════════════════════════════════════════════════════════
INPUT 2 — VIDEO PLACEMENTS (video_placements.json):
{video_placements_json}

═══════════════════════════════════════════════════════════════════════
TASK:

1. MAP each video placement action to the correct manual step it belongs to.
   - Use part descriptions and the visual progression of the build to determine which
     manual step each placement corresponds to.
   - A single manual step will typically contain one or more consecutive placement actions
     (sub-steps).
   - If a placement cannot be confidently mapped to any step, assign it to the nearest step.

2. CREATE ordered micro sub-steps within each manual step.
   - Sub-steps are numbered sequentially: 1.1, 1.2, 1.3 ... for step 1, etc.
   - Each sub-step maps to one placement action from the video.
   - Sub-steps describe exactly what the user does at each individual placement.

3. CORRECT errors in the manual where video evidence clearly contradicts it.
   - Only flag a correction when there is strong video evidence (e.g., clearly different
     color, wrong part size, wrong quantity).
   - Do NOT correct if the difference is likely just naming convention or ambiguity.
   - Each correction must specify the field, original value, corrected value, reason,
     and confidence.

═══════════════════════════════════════════════════════════════════════
Return ONLY valid JSON in this exact format:

{
  "steps": [
    {
      "step_number": 1,
      "sub_steps": [
        {
          "sub_step_number": "1.1",
          "action_description": "Add a red circular 2x2 brick to the bottom left of the grey baseplate",
          "placement_index": 0,
          "parts_involved": ["red round tile"],
          "confidence": 0.9
        },
        {
          "sub_step_number": "1.2",
          "action_description": "Add a white 2x2 brick directly to the right of the red tile",
          "placement_index": 1,
          "parts_involved": ["white 2x2 brick"],
          "confidence": 0.85
        }
      ],
      "corrections": [
        {
          "field": "parts_required",
          "original_value": "red 2x2 brick",
          "corrected_value": "red round tile",
          "reason": "Video clearly shows a round tile with circular profile, not a standard square 2x2 brick",
          "confidence": 0.88
        }
      ]
    }
  ]
}

RULES:
- Include ALL manual steps in the output, even if they have no sub-steps (use empty arrays).
- placement_index refers to the index in the video_placements.json placements array.
- corrections is an empty array if no errors are found for that step.
- Only include corrections with confidence > 0.7.

Do NOT include any text before or after the JSON.
