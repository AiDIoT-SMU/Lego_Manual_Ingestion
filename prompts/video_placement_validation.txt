You are analyzing consecutive placement frames from a LEGO assembly video to identify what new part was added.

**YOUR TASK:**

Compare the PREVIOUS PLACEMENT FRAME with the CURRENT PLACEMENT FRAME and determine:
1. Look at the CURRENT frame and identify every LEGO brick that is physically snapped onto studs (ignore hands, background, the table surface)
2. Look at the PREVIOUS frame and do the same count — only bricks snapped onto studs, ignoring hands
3. If the CURRENT frame has a brick that is clearly attached to the assembly and was NOT present in the PREVIOUS frame, report it as a new part
4. If you are not sure whether what you see is a brick or a hand/shadow/part of the baseplate, do NOT report it

**ANALYSIS GUIDELINES:**

**Part Identification:**
- Describe the part with: COLOR + TYPE + SIZE (e.g., "red 2x4 brick", "white round 2x2 brick")
- Only identify a part after you have observed it in the frame
- Use the expected_parts_list to help name it accurately, but only if you already see it
- If the part matches expected parts, set matched_from_manual: true
- If the part doesn't match expected parts, still describe it accurately but set matched_from_manual: false

**CRITICAL - How to accurately determine brick dimensions:**
- COUNT THE STUDS (round bumps on top) carefully to determine size
- A 1x2 brick has 2 studs in a single row (rectangular, longer shape)
- A 2x2 brick has 4 studs in a 2x2 grid (square shape)
- A 2x4 brick has 8 studs in a 2x4 grid (rectangular, larger)
- Examine the brick's shape: rectangular vs square, tall vs flat
- Common mistake: confusing 1x2 (rectangular, 2 studs) with 2x2 (square, 4 studs)

**Spatial Positioning:**
- **location**: Where on the assembly (e.g., "left side", "center", "top", "front edge")
- **reference_object**: What is it attached to? (e.g., "baseplate", "previous white brick", "center tower")
- **orientation**: How is it oriented? (e.g., "horizontal", "vertical", "upright")
- **relative_position**: If applicable, describe position relative to other parts (e.g., "to the right of the red brick")

**Duplicate Detection:**
- A frame is a duplicate when the assembly has the same attached bricks in the same configuration as the previous frame
- Look carefully at every visible part — one additional brick in a row means a new part was added
- If a hand is obscuring part of the assembly, do NOT assume a brick is hidden underneath — only count what is clearly visible
- Background objects that aren't part of the active assembly should be ignored

**Subassembly Handling:**
- If the user is working on a separate subassembly (different from the main build), note it
- Set is_subassembly_switch: true if the user switched to a different assembly area
- Ignore parts in the background that aren't being actively assembled

**Bounding Box Localization:**
- If you claim `has_new_part: true`, you MUST provide a `box_2d` that tightly encloses the newly added part in the CURRENT PLACEMENT FRAME
- Use the format `[ymin, xmin, ymax, xmax]` where each value is an integer from 0 to 1000, representing the position as a fraction of the frame's height/width (0 = top/left edge, 1000 = bottom/right edge)
- Example: a part in the bottom-left quarter of the frame might be `[600, 50, 900, 300]`
- If you are unable to precisely locate the newly added part in the CURRENT PLACEMENT FRAME, set `confidence` below 0.5 to indicate low certainty
- If you set `has_new_part: false`, set `box_2d` to null
